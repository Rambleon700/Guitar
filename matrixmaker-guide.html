Matrix Maker: Full How-To

Matrix Maker builds custom strum-and-chord loops with realistic plucked-string audio, humanized groove, a live fretboard view, a practice-tempo engine, and full save/export support. This guide walks through every capability, start to finish.

Quick Start

The fastest way to hear something:

  1. The workspace loads with two starter blocks, Am and C, already in place.
  2. Click ▶ Run Matrix Loop (top of the page, under the fretboard, or the sticky bar pinned to the bottom of the screen — all three do the same thing).
  3. You'll hear a 1-bar count-in click, then the loop plays: each chord strums through its pattern and the fretboard, ribbon, and big arrow all animate in sync.
  4. Click ■ Stop Loop any time.

Everything below is about customizing that loop — which chords, which strums, what groove and tempo, and how to save or share what you build.

Building Chord Blocks

Each block in the Matrix Grid represents one chord shape and its strum pattern. You can add blocks two ways:

  • Load From Open Library — pick a common chord (open-position majors/minors, common barre shapes) from the dropdown; it's added instantly with correct fret placement.
  • + Append Custom Block — adds a blank block with all strings muted (X) so you can build any shape from scratch.

Editing frets

Each block shows six fret fields, one per string (low E through high e). Type into any field:

ValueMeaning
XString is muted / not played
0Open string
1–15Fretted note at that fret number

Changes apply instantly to the fretboard preview, so you can dial in a shape by ear and by eye before adding it to the loop.

Naming and repeats

The text field at the top of each card is the chord's display name (shown in the ribbon track and status strip). Next to it, Repeat ×N controls how many times that block's full strum pattern plays before the loop moves to the next chord — set it to 2, 3, or 4 to hold a chord for multiple bars, exactly like a real song's phrasing.

Click Delete on any card to remove it from the sequence.

Strum Patterns

Below each chord's fret grid is its Strum Patterns Blueprint — a row of tokens showing exactly what happens on every stroke in that chord's cycle:

⬇ D = Downstroke   ⬆ U = Upstroke   ✕ M = Missed Beat   ● P = Chuck (percussive mute)

Each block can have its own unique pattern — a slow ballad chord might just be D, while a strummy chorus chord might be D D U U D U.

Missed beats (M)

A missed beat still takes up its full slot in the timing — your strumming hand keeps moving on the beat — but no strings are actually struck, so it plays silent. For example D D U M U D U skips the string contact on one upstroke while keeping the rhythm hand moving evenly.

Chuck / percussive mute (P)

Different from a Missed beat: a Chuck is an audible short, low, percussive "thunk" — like resting your strumming hand on the strings to mute them and hitting them anyway. It's the classic muted-strum sound used in funk, pop, and reggae-adjacent rhythm parts. Add it via the + Chuck (P) button in the Pattern Bank builder (see below).

Accenting individual strokes

Click any token in a chord's pattern (not the × remove button) to toggle an accent on that specific stroke — it gets a highlighted glow and plays louder than its neighbors, independent of the global Turn 2 accent setting. This is how you emphasize a specific beat in a bar, exactly like a real strumming hand naturally does.

If a block has no strums defined, it defaults to a single downstroke.

🔨 Strumming Pattern Blueprint Bank

Instead of rebuilding the same strum pattern on every chord, build it once and save it for reuse:

  1. Name the pattern (e.g. "Pop", "Waltz", "Reggae Skank") in the text field.
  2. Click + Down (D) / + Up (U) / + Missed (M) / + Chuck (P) repeatedly to build the sequence — the preview box shows exactly what you're building.
  3. Click 💾 Save Pattern to store it. It appears as a chip below, with a small [⬇ File] link to export just that one pattern to disk.
  4. Click Clear Work to reset the builder without saving.

Saved patterns are stored in your browser and can be injected directly into any chord block's strum area from that block's inject-target buttons. Once injected, you can still click individual tokens on the chord card to add accents.

Backing up your pattern bank

Your whole pattern bank can be exported to a .json file with ⬇ Export Bank, and restored (or merged onto another device/browser) with ⬆ Import Bank. If an imported pattern name already exists, you'll be asked whether to overwrite it.

📂 Persistent Full-Loop Archive

This saves your entire workspace — every chord block, its frets, its strum pattern (including accents), its repeat count, and the tempo — as one named entry.

  1. Type a name (e.g. "Verse Progression", "Song Intro") into the field.
  2. Click 💾 Archive Whole Workspace Layout.
  3. It appears below as a chip with [Load] (restores it into the workspace, replacing what's there) and [⬇ File] (exports just that one loop).

⬇ Export All Loops downloads your whole archive as one .json file — a full backup. ⬆ Import Loops restores from that file, merging into whatever's already saved (with an overwrite prompt on name collisions). This is the way to move your archive between devices or back it up somewhere safe.

Clear Entire Archive deletes everything in storage — export a backup first if you want to keep it.

Playback Engine

ControlWhat it does
Tempo50–150 BPM manually (Tempo Ramp, below, can push it higher automatically).
Drum Grid TrackMetronome Click (simple click on beat 1, ticks elsewhere) or 4/4 Standard Rock (kick / snare / hi-hat pattern).
Turn 2 Dynamic AccentuationWhen enabled, every other pass through the full sequence plays with a louder, brighter accent — useful for hearing the difference between a soft verse strum and an accented chorus strum.

The chord audio itself uses a plucked-string synthesis engine (not a simple beep) with natural decay, a warm body-resonance tone, and a touch of room reverb — it's meant to sound like a real strummed acoustic, not a toy. Chord tails naturally ring into the next chord's onset instead of being cut off, just like a real strum.

🏋️ Practice Mode

Count-in

Enabled by default. Before the loop starts, you'll hear one bar (4 clicks) with a visible "Count-in: 4…3…2…1" readout, so you always come in on the beat.

Tempo Ramp

Turn this on to have the tempo automatically increase as you complete full loop passes — the classic "practice slow, build to full speed" method:

SettingMeaning
+BPMHow many BPM to add each time the ramp triggers.
Every N LoopsHow many complete passes through the whole sequence before the tempo bumps up.
Cap AtThe ramp will never push the tempo past this ceiling.

The tempo display updates live as the ramp fires, so you can see your practice tempo climbing in real time.

🎼 Groove & Realism

These settings shape how "human" the loop feels, beyond just which notes play:

SettingWhat it does
Humanize TimingOn by default. Adds a small random timing offset (± the ms you set) to every strum and drum hit, instead of everything landing on a perfectly rigid grid.
SwingOff by default. Offsets alternating strokes long-short for a shuffled feel (adjust the Swing % — 50% is straight, higher numbers swing harder). Total loop length stays the same either way.
Realistic UpstrokesOn by default. Automatically limits upstrokes to the top strings (when they're fretted) instead of ringing the full chord — matching how a real strumming hand naturally catches fewer strings on the way up.
Attack TonePick (bright, sharp attack, the default) or Fingers (mellower, softer attack) — changes the character of every note in the loop.

Underneath the hood, every pluck also gets a small amount of automatic micro-variation in tone and timing — so the same chord struck twice in a row never sounds perfectly identical, the way a real instrument doesn't either.

Reading the Live View

While the loop plays, several things update together:

  • Fretboard — shows exactly which frets are held for the currently sounding chord.
  • Color Coded Ribbon — a horizontal timeline of every block in your sequence; the currently playing block lights up so you can see where you are at a glance.
  • Big Strum Arrow — a large pulsing symbol directly above the ribbon showing exactly what just happened: ⬇ amber for down, ⬆ blue for up, ✕ grey for a missed beat, and ▪ red for a chuck. An extra glow means that stroke was accented.
  • Status Strip — text readout of the current step (with repeat progress like "2 / 4 (rep 1/2)" when a chord repeats), which loop pass you're on, engine status (accent on/off), and the exact fret values sounding right now.
  • Sticky Transport Bar — a slim strip pinned to the bottom of the screen with its own Run/Stop buttons and loop counter, always reachable no matter how long your sequence gets or how the page is embedded.

🎧 Audio Export & Sharing

Export Audio (WAV)

Set Loops to Render (1–8) and click ⬇ Export Audio (WAV). This renders your current workspace offline — same engine, same tone, same tempo/drum/accent/repeat/groove settings — into a real downloadable .wav file. Because it's rendered offline rather than captured live, there's no dropped audio or timing jitter, even at high loop counts.

Copy Share Link

Click 🔗 Copy Share Link to pack your entire workspace (chords, frets, strums with accents, repeats, tempo, drum style, accent setting) into a URL and copy it to your clipboard. Anyone who opens that link gets your exact loop pre-loaded — no file to send, just paste the link.

Share links and exported archive files both carry the full sequence — the difference is a share link is a one-click URL, while an archive file is something you keep and re-import later.

⚠ Saving & Embedding Notes

Matrix Maker saves patterns and loops to your browser automatically as you work. In most cases this "just works" — but there's one situation worth knowing about:

Embedded pages (e.g. a Google Sites gadget) can block browser storage entirely. Some embed methods run the page inside a sandboxed frame that doesn't allow saving data at all. If that happens, Matrix Maker detects it automatically and shows a warning banner at the top of the page — everything still works normally for that session, but nothing will survive a page refresh.

If you see that warning (or just want a guaranteed-safe backup no matter where you're using the tool):

  • Use ⬇ Export Bank and ⬇ Export All Loops right after building something you want to keep.
  • Keep those .json files somewhere safe (or in your GitHub repo alongside the tool itself).
  • ⬆ Import them back any time, on any device or browser — this always works regardless of storage restrictions.

For guaranteed persistent in-page saving, host the file directly (e.g. GitHub Pages) rather than embedding it through a gadget that sandboxes storage.
